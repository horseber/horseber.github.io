JVM内存模型(JDK8)

发表于 2020-06-21 更新于 2020-09-19 分类于 JVM
JDK8的jvm数据区内存结构图1.8同1.7比，最大的差别就是元数据区取代了永久代。

JDK8的jvm数据区内存结构图：

1.8同1.7比，最大的差别就是：元数据区取代了永久代。

元空间的本质和永久代类似，都是对JVM规范中方法区的实现。不过元空间与永久代之间最大的区别在于：

  • 元数据空间并不在虚拟机中，而是使用本地内存。

  • 字符串常量池从永久代中剥离出来，存放在堆中

  • 永久代必须限制大小，元数据空间无上限

  • 1、程序计数器

  • 2、java虚拟机栈

  • 3、本地方法栈

  • 4、堆

1.8后，字符串常量池从永久代中剥离出来，存放在队中。堆有自己进一步的内存分块划分

  • 5、元数据区

元数据区取代了1.7版本及以前的永久代。元数据区和永久代本质上都是方法区的实现。方法区存放虚拟机加载的类信息，静态变量，常量等数据。
