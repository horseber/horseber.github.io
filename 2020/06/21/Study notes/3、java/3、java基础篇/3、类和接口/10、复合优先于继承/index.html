复合优先于继承

发表于 2020-06-21 更新于 2020-09-19 分类于 类和接口
一继承的缺陷1、打破封装原则

一：继承的缺陷：

  • 1、打破封装原则：

继承使得子类依赖于超类的实现，从这一点来说，就不符合封装的原则。

  • 2、子类代码容易受到破坏：

子类继承的超类随着版本的发布而有所变化，子类就有可能遭到破坏，即使子类自己的代码完全没有改变。

如果父类新增了方法，而子类恰好已经提供了一个签名相同但是返回值不同的方法，那么子类将无法通过编译。
